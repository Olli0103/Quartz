### ITSM Integration Service Status

- Service scope clearly defined but limited
  - Integrates customer ITSM systems (ServiceNow, Jira) to Cloud ALM via standard API
  - Customer’s ITSM system integration remains customer responsibility
  - CAS only handles Cloud ALM to Support Backbone integration
- Current positioning strategy
  - Not marketed as separate service
  - Derivative of Application Management
  - Only offered in larger agreements
  - Deliberately not actively promoted

### Service Limitations and Risks

- Integration supports incidents only (not service requests)
  - Risk: customers might route service requests through ITSM instead of Service Request App
  - Could undermine automation and impact margins
  - Threatens existing ECS workflow efficiency
- Initial setup: 2-day standard API configuration
- Ongoing monitoring and customization included
