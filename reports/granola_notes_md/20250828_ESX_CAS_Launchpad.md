### ESX CAS Launchpad Migration Overview

- Long-term goal: migrate current CAS launchpad to new ESX isolated platform
  - ESX owned entirely by team (no ECS sharing)
  - Full control over access, costs, entitlements
- Current launchpad connected to uncontrollable FAC global account
  - Wild growth over 10+ years
  - Shared with ECS, difficult cost/entitlement control

### Technical Architecture & Approach

- Follow S4HANA public cloud approach: maximum ABAP Cloud usage
  - ABAP Cloud for UI layer (more stable, robust)
  - Build Work Zone on top for Fiori launchpad entry point
- Publishing mode implementation required
  - Backend (BTP ABAP Cloud) publishes roles to Build Work Zone
  - Apps run in ABAP while tiles generated by Build Work Zone
  - Communication scenario/arrangement setup needed

### Security Applications & Roles

- Initial focus: security update process management apps
  - Move from SharePoint/Teams processes to proper Fiori apps
  - 3-4 apps being built for application security updates
- Two primary roles required:
  - “Application Security Updates - Read Only” (CAS delivery)
  - “Application Security Updates - Manage”
- ARM integration mandatory for role requests/approvals
  - External colleagues will access system
  - Formal approval process required

### Implementation Timeline & Priorities

- Target: November 2024 (flexible, as early as possible)
- Onboard 5-10 CAS delivery security team members
- Phase 1: Production setup with apps and ARM roles
  - Start directly in production (no current users, minimal risk)
  - Identity provider configuration needed
- Phase 2: Fiori launchpad activation (January 2025)
