### IBM Project Status & Challenges

- Contract signed for BTP, CALM, and Advanced Security packages
  - Senior resources deployed: Raquel (CALM), Viani (Security), Pradhan (Cloud Integration), André/Fernando (BTP)
  - Customer expects SAP to show “best run business” using their existing tools (SignalVio, CVA, ALM, ATC, LeanIX)
  - Initial scope creep when customer demanded comprehensive tool integration guidance
- Current Q1 deliverable: Clean Core health score for DSW system
  - Focus on extensibility, operations, and integration dimensions
  - Run reports (ATC, SCCN, USG), classify code A-D, present in CALM
  - Establish transparency dashboard for operations visibility
- Infrastructure complexity
  - Premium supplier contract separate from SAP services
  - Application operations, security updates, RVU handled by IBM infrastructure
  - Customer refuses basis management from SAP
  - Foundation migration blocked without basis access

### Governance & Collaboration Model

- Three-way alignment established with SAP stakeholders
  1. Enterprise Architect (owns Clean Core and integrated tool chain)
  2. TQM from PE
  3. BTP Customer Success Manager
- Weekly Wednesday standup meetings for integrated status reporting
- EA leads customer presentations and owns certain tool implementations
- Challenge: Customer spend $1B on SAP but small SAP services contract creates expectation mismatch

### Lessons for Scalable Delivery

- Need integrated messaging vs siloed service presentations
  - Avoid concatenated slide decks per service
  - Focus on outcomes, not activity metrics (e.g., “reduced 20% alerts YoY” vs “closed 590 of 600 alerts”)
- Required quick wins for smaller customers
  - Basic service plans maintained in CALM
  - Standardized reporting templates
  - Minimum viable collaboration model with MAX/CSMs
- Automation showcase needed
  - Highlight inherent efficiencies and automated processes
  - Address customer question: “Does cloud integration testing ever fail or is it always green?”
