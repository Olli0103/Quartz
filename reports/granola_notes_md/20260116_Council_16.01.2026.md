### PAM (Proactive Application Management) Strategy Discussion

- Oliver raised concerns about PAM implementation quality

  - Standard client ANN delivery is insufficient
  - Private Cloud: comprehensive monitoring capabilities available
  - SuccessFactors: monitoring capabilities “mitteldünn” (medium-thin)
  - BTP: some capabilities available but issues with native BTP customers
  - Ariba/Sales Performance Management: severely limited useful KPIs

- Three core problems identified:

  1. Product capability gaps across different SAP solution areas
  2. Delivery capacity constraints
  3. Integration methodology not fully embedded in delivery approach

### SuccessFactors Fixed Package Monitoring

- Monitoring included as mandatory component in Fixed Package

  - Cloud ALM provides standard KPIs for SuccessFactors
  - Generates alerts → Service Desk email → incident management process
  - Alternative to full 24/7 monitoring (too expensive for lower-tier offerings)

- Business justification for keeping monitoring:

  - Essential for “Easy to sell” messaging
  - Provides proactive value beyond reactive ticket handling
  - Customer expectation for SAP operations services
  - Without monitoring: “naked” service offering insufficient differentiation

### BTP Operations Service Structure

- Three vertical focus areas for BTP tailored edition:

  1. Application Management (ticket-based, outcome-based operations)
  2. Development environment support (customer-built apps, monitoring alerts)
  3. Regression Suite Testing (anomaly - never sold)

- Current monitoring approach:

  - F-Ran system for monitoring (problems with native BTP customers)
  - Need customer to exist in private environment
  - Exploring Cloud ALM as alternative platform

### Vision for Integrated Application Management

- Cross-solution architecture proposal:

  - Reactive component: Security, Performance, Data ILM, AI Operations
  - Proactive component: Cloud ALM monitoring, dashboarding, reporting
  - Clean Core/Clean Code integration for ERP Private + BTP scenarios
  - Unified ticketing system integration

- Timeline considerations:

  - Current: Q1 2026 implementation target
  - Pre-approved packages needed for scalable delivery
  - Easy attachment model similar to cloud business contracts
